# 地铁人流量预测 - 项目记忆

## 赛题概述

- **目标**：预测未来一天各站点以10分钟为单位的进站(inNums)和出站(outNums)人次
- **训练数据**：2019-01-01 至 2019-01-25，共25天，约7000万条刷卡记录
- **线路**：3条线路（A/B/C），81个地铁站（stationID 0-80，54号缺失）
- **评估指标**：MAE（入站MAE + 出站MAE的平均值）
- **建模范围**：仅 B 线（站点 0-33，34站），Closed-loop 设定

## 数据字段说明

| 字段      | 含义                       |
| --------- | -------------------------- |
| time      | 刷卡时间                   |
| lineID    | 线路ID (A/B/C)             |
| stationID | 站点ID (0-80)              |
| deviceID  | 设备ID                     |
| status    | 0=进站, 1=出站             |
| userID    | 用户ID (payType=3时非唯一) |
| payType   | 支付类型 (3=非唯一用户)    |

## 预测输出格式

- 每10分钟一个时间段：[startTime, endTime)
- 需预测：inNums（进站人数）, outNums（出站人数）
- 示例：`0,2019-01-29 00:40:00,2019-01-29 00:50:00,100.0,200.0`

### Q&A：建模关键问题

#### 1. 各站点是分开预测还是联合预测？

**联合预测**。所有 34 个站点放在同一个模型中训练，`stationID` 作为 One-Hot 特征输入。这样模型能利用跨站点的共性模式（如早晚高峰规律），同时通过 One-Hot 捕捉站点个性差异。进站和出站分别训练两个独立模型（XGBoost × 2）。

#### 2. 全线 R² 多少？

| 指标          | 进站             | 出站             | 平均             |
| ------------- | ---------------- | ---------------- | ---------------- |
| **R²** | **0.9761** | **0.9806** | **0.9784** |
| MAE           | 18.69            | 15.13            | 16.91            |

R² 接近 0.98，说明模型解释了约 98% 的流量方差，拟合非常好。

#### 3. 每个站点的 R² 多少？

- **进站 R²**：范围 [0.8192, 0.9821]，均值 0.9258
- **出站 R²**：范围 [0.9148, 0.9877]，均值 0.9667
- **全部 34 站 R² 均为正**，无一负值
- 出站预测普遍优于进站（出站均值 0.967 > 进站均值 0.926）
- 表现最好的站点：站点 4（R²_avg=0.983）、站点 5（0.979）
- 表现最弱的站点：站点 24（R²_avg=0.882）、站点 28（0.887）

详见下方完整站点 R² 表。

#### 4. 训练中是否考虑了站点间的连接关系（路网图）？

**没有**。当前模型仅将 `stationID` 作为分类特征（One-Hot Encoding），未使用 `Metro_roadMap.csv` 中的邻接矩阵信息。这是一个明确的改进方向——可以将相邻站点的流量作为特征（空间滞后），或使用图神经网络（GNN）建模站点拓扑。

#### 5. 评价指标是什么？

- **赛题指标**：MAE（Mean Absolute Error），分别对进站和出站计算 MAE 后取平均
  $$
  \text{Score} = \frac{1}{2}\left(\text{MAE}_\text{in} + \text{MAE}_\text{out}\right)
  $$
- **本项目使用**：与赛题一致的 MAE。另外也计算了 R² 作为参考
- **本项目 XGBoost 成绩**：MAE_in=18.69, MAE_out=15.13, MAE_avg=16.91

## GNN 改进方向

### 为什么用 GNN？

当前模型将 34 个站点视为**彼此独立的特征**（One-Hot）。但实际上：

- 站点 5 的早高峰客流 → 10 分钟后会出现在相邻站点 6 的**出站**口
- 站点 15（枢纽站）的异常客流会**传导**到整条线的上下游
- 路网图 `Metro_roadMap.csv` 已经给出了站点连接关系，但没有被利用

GNN 的核心优势：让相邻站点的信息在训练中**互相传递**，学到"这个站人多，下游站 20 分钟后也会人多"的空间依赖。

### GNN 改进流程（以 ST-GCN 为例）

```
┌─────────────────────────────────────────────────────┐
│  Step 1  图构建                                      │
│  ├─ 节点: 34 个 B 线站点                             │
│  ├─ 边: Metro_roadMap 邻接矩阵 (0-1)                 │
│  ├─ 节点特征: 每时段 inNums/outNums + 时间特征        │
│  └─ 输出: 图序列 G₁, G₂, ..., Gₜ (每个时段一张图)     │
├─────────────────────────────────────────────────────┤
│  Step 2  空间特征提取 (GCN / GAT)                     │
│  ├─ 对每张图 Gₜ，用 GCN 层聚合邻居节点信息             │
│  ├─ 公式: H' = σ(D⁻½ A D⁻½ H W)                     │
│  ├─ 含义: 站点 i 的新特征 = 自己 + 邻近站点的加权平均  │
│  └─ 输出: 每个站点获得融合了邻居信息的嵌入向量          │
├─────────────────────────────────────────────────────┤
│  Step 3  时序建模 (GRU / TCN)                         │
│  ├─ 将 Step 2 的图嵌入按时间排列: [E₁, E₂, ..., Eₜ]  │
│  ├─ 输入 GRU/TCN 捕捉"前几个小时影响后几个小时"的规律   │
│  └─ 输出: 未来时段的预测值                             │
├─────────────────────────────────────────────────────┤
│  Step 4  训练与预测                                   │
│  ├─ 损失函数: MAE / MSE                               │
│  ├─ 输入: 过去 N 个时段的图序列                        │
│  └─ 输出: 未来 M 个时段的各站点 inNums/outNums         │
└─────────────────────────────────────────────────────┘
```

### 关键技术选型

| 组件   | 方案 A（简单）                    | 方案 B（进阶）                                |
| ------ | --------------------------------- | --------------------------------------------- |
| 图卷积 | **GCN** (Graph Convolution) | **GAT** (Graph Attention，自动学边权重) |
| 时序   | **GRU** (门控循环单元)      | **Temporal ConvNet** (TCN)              |
| 框架   | **PyTorch Geometric** (PyG) | **DGL** (Deep Graph Library)            |

### 可尝试的模型架构

**方案一：ST-GCN（时空图卷积，最简单入门）**

```
输入 [T, N, F]  →  GCN(空间) → GRU(时间) → FC → 输出 [N, 2]
```

> N=34站, T=历史时段数(如12个=2小时), F=特征数

**方案二：Graph WaveNet（2019, 效果更好）**

```
输入 → 扩散图卷积 + 膨胀TCN → 自适应邻接矩阵 → 输出
```

> 优点：自动学习站点间隐含的连接关系（不限于路网给定的边）

**方案三：ASTGCN（注意力时空图卷积）**

```
输入 → 空间注意力 + 时间注意力 + GCN → 多组件融合 → 输出
```

> 优点：同时学习"哪些时段重要"和"哪些邻居站点重要"

### 与当前 XGBoost 的关系

- GNN **不是替代** XGBoost，而是**补充空间特征**
- 可以将 GCN 提取的"站点嵌入向量"作为新特征，**拼回 XGBoost 训练**
- 这是一种**混合方案**：GNN 负责空间 → XGBoost 负责最终回归
- 预期提升：5%~15%（尤其改善当前 R² 较低的那些边缘站点）

### 参考论文

- ST-GCN: *Spatial Temporal Graph Convolutional Networks* (Yan et al., 2018)
- Graph WaveNet: *Graph WaveNet for Deep Spatial-Temporal Graph Modeling* (Wu et al., 2019)
- ASTGCN: *Attention Based Spatial-Temporal Graph Convolutional Networks* (Guo et al., 2019)

### 附：LSTM / GRU / GCN / Graph WaveNet / ASTGCN 区别

这几个名字容易混淆，本质区别在于**处理空间还是时间**：

```
                    只处理时间           只处理空间         同时处理时空
                    ─────────           ─────────         ────────────
有门控             LSTM, GRU                               Graph WaveNet
                                                        (Gated TCN 部分)
无门控 / 其他                                        GCN          ASTGCN
                                                  (纯图卷积)    (注意力+图卷积)
```

#### 详细对比

| 模型 | 处理维度 | 核心机制 | 门控？ | 一句话 |
|------|:---:|------|:---:|------|
| **LSTM** | ⏳ 时间 | 遗忘门+输入门+输出门 | ✅ 三门 | "记住有用的历史，忘掉无关的" |
| **GRU** | ⏳ 时间 | 重置门+更新门 | ✅ 两门 | "LSTM 简化版，更快，效果差不多" |
| **GCN** | 🕸️ 空间 | 邻居特征加权平均 | ❌ 无门 | "我的特征 = 邻居们的平均值" |
| **Graph WaveNet** | ⏳🕸️ 时空 | 扩散GCN + 门控TCN | ✅ 部分 | "GCN做空间 + 门控卷积做时间" |
| **ASTGCN** | ⏳🕸️ 时空 | 注意力 + GCN + 时间卷积 | ❌ 用注意力替代门控 | "自动学习哪些邻居/时段更重要" |

#### 逐步理解

**第一层：LSTM vs GRU（只做时间，没有图）**
```
LSTM:  记忆细胞 cₜ 缓缓变化（长期记忆）
       隐藏状态 hₜ 快速响应（短期输出）
       三个门控制：忘什么、记什么、输出什么
       
GRU:   把 cₜ 和 hₜ 合并成一个状态
       两个门控制：重置（看多少历史）、更新（保留多少旧信息）
       → 参数更少，训练更快，效果接近 LSTM
```

**第二层：GCN（只做空间，没有时间）**
```
GCN:   站点 i 的新特征 = 站点 i 自己 + 所有邻居站点的平均
       没有"门"，就是一次加权求和
       相当于：把路网图的信息"扩散"一步
       
类比:  你做小组作业，GCN 就是你把组员的意见平均一下
       LSTM/GRU 是你自己回忆上周做了什么
```

**第三层：Graph WaveNet（时空结合，有门控）**
```
结构:  空间层（扩散GCN）→ 时间层（门控TCN）→ 输出

时间层的门控 TCN:
  输入 x → 分两路:
    A 路: tanh(卷积(x))    ← 候选值
    B 路: sigmoid(卷积(x))  ← 门控（0~1之间）
  输出 = A ⊙ B              ← 门控决定候选值通过多少

→ 这里的"门"和 LSTM/GRU 的门是同一个思想：
  让网络自己学习"这个时段的信息重不重要"
```

**第四层：ASTGCN（时空结合，用注意力替代门控）**
```
结构:  空间注意力 → GCN → 时间注意力 → 输出

注意力 vs 门控:
  门控:    用 sigmoid 输出 0~1，决定"信息通过多少"
  注意力:   用 softmax 给所有位置打分，加权求和
  本质区别: 门控是"过滤"，注意力是"选择关注谁"
  
ASTGCN 的特别之处:
  - 空间注意力: "预测站点5时，站点6比站点20更重要"
  - 时间注意力: "早上8点比凌晨3点更值得参考"
  → 这些权重是模型自动学出来的，不是人定的
```

#### 对本项目的适用性

| 模型 | 能用吗 | 说明 |
|------|:---:|------|
| LSTM/GRU | ✅ 已隐含 | 滞后特征 `lag_1d/7d` 间接捕捉了时序依赖 |
| GCN | 需新增 | 需要引入 `Metro_roadMap` 邻接矩阵 |
| Graph WaveNet | ✅ 推荐 | 门控 TCN 比 GRU 更快，扩散 GCN 适合地铁拓扑 |
| ASTGCN | ✅ 进阶 | 注意力权重可以做可视化，"展示价值"高 |

> 可以直接展示"模型学会了早高峰时站点5最关注站点4和站点6"。
